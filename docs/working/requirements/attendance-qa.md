# 勤怠管理アプリケーション — 要求仕様 Q&A

## 1. ユーザーと利用シーン

[Question] このアプリの利用者は誰ですか？（例: 一般社員のみ / 社員＋管理者 / 社員＋管理者＋人事）
[Answer]社員＋管理者＋人事

[Question] 想定する利用人数の規模はどのくらいですか？（例: 〜50人 / 〜500人 / 1000人以上）
[Answer]500

[Question] PC ブラウザ利用のみですか？スマートフォン対応も必要ですか？
[Answer]PC　ブラウザのみ

勤怠一覧の表示（日別 / 月別）

## 3. 勤務体系

[Question] 勤務形態はどれを想定していますか？（複数選択可）

[Question] 所定労働時間は何時間ですか？（例: 8時間 / 7.5時間）

[Question] 残業の計算ルールはありますか？（例: 所定超過分を残業 / 深夜割増あり / 月45時間上限アラート）

## 追加確認

[Question] 「承認者」と「承認される側」の2ロールとのことですが、「人事」は別ロールとして必要ですか？（例: 全社員の勤怠閲覧・社員マスタ管理など）それとも管理者＝承認者＝人事で同一ですか？
[Answer]人事ロールは作る、でも勤怠の通常の上長的な承認者と部下として承認を受ける立場はある
人事ロールは変わりに全員の承認権月と全員の勤怠が見れる

[Question] 承認フローは「月単位でまとめて承認」ですか？「日ごとに承認」ですか？
[Answer]月単位だけど一覧を見て承認

[Question] 休憩は固定1時間（12:00〜13:00）とのことですが、打刻は不要で自動控除でよいですか？
[Answer]はい

[Question] 承認者は「自分の部下だけ」承認できる想定ですか？（＝部署やチームの概念が必要？）それとも MVP では承認者は全員の勤怠を承認できる形でよいですか？
[Answer]はい

[Question] 社員マスタ（社員の登録・編集・削除、ロール割り当て）は人事ロールが画面から行いますか？それとも初期はDB直接投入やシード等でよいですか？
[Answer]はい
