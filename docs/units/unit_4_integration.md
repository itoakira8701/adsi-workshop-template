# Unit 4: 統合テスト

## 目的

全 Unit が結合した状態でエンドツーエンドの動作を確認する。

## 前提

- Unit 0〜3 すべて完了済み

## スコープ

### Backend 統合テスト

- 認証 → 打刻 → 一覧確認 → 承認の一連フロー
- ロール別アクセス制御の横断テスト
- 異常系（権限なし、二重打刻、未出勤退勤 等）

### Frontend E2E テスト（Playwright）

- ログイン → 打刻 → 勤怠一覧確認
- 承認者ログイン → 承認一覧 → 承認実行
- 人事ログイン → 社員登録 → ロール割り当て
- 権限なしページへのアクセス → リダイレクト確認

### 全体確認

- SageMaker プレビューでの動作確認
- パフォーマンス確認（500名規模のシードデータ）
