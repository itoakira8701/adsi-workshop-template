# Unit of Work 分割

## 依存図

```
Phase A（共通基盤 — 1人で先行）
┌─────────────────────────────────────┐
│  Unit 0: 共通基盤                    │
│  プロジェクト骨格・認証・Entity・DB   │
└──────────────┬──────────────────────┘
               │ 依存
       ┌───────┴───────┐
       │               │
       ▼               ▼
Phase B（2人で並行実装）
┌─────────────────┐  ┌─────────────────┐
│ Unit 1: 打刻     │  │ Unit 2: 社員     │
│ 勤怠記録ドメイン  │  │ マスタ管理       │
└────────┬────────┘  └─────────────────┘
         │
         ▼
Phase C（Unit 1 完了後）
┌─────────────────┐
│ Unit 3: 月次承認 │
└─────────────────┘

Phase D（全 Unit 完了後）
┌─────────────────┐
│ Unit 4: 統合     │
│ E2E・結合テスト   │
└─────────────────┘
```

## Phase 割り当て

| Phase | Unit | 担当 | 前提 |
|-------|------|------|------|
| A | Unit 0: 共通基盤 | 合同 | なし |
| B | Unit 1: 打刻・勤怠一覧 | 担当者 A | Unit 0 |
| B | Unit 2: 社員マスタ管理 | 担当者 B | Unit 0 |
| C | Unit 3: 月次承認 | 担当者 A or B | Unit 0 + Unit 1 |
| D | Unit 4: 統合テスト | 合同 | 全 Unit |

## 並行実装ポイント

- Unit 1 と Unit 2 は **Employee Entity を共有**するが、それは Unit 0 で定義済み
- Unit 1 は attendances テーブル、Unit 2 は employees テーブルの CRUD → 衝突しない
- Unit 3 は勤怠データの存在が前提なので Unit 1 の後

## 各 Unit 概要

| Unit | ユーザーストーリー | テーブル | API |
|------|-----------------|---------|-----|
| 0 | — | employees（スキーマのみ） | /auth/login, /auth/me |
| 1 | 打刻・勤怠一覧表示・勤務時間計算 | attendances | /attendances/* |
| 2 | 社員登録・編集・削除 | employees（CRUD） | /employees/* |
| 3 | 月次承認一覧・承認実行 | monthly_approvals | /approvals/* |
| 4 | 全体結合 | — | — |
