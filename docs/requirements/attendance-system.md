# 勤怠管理システム — 要求仕様書

## 概要

社員の出退勤を管理するWebアプリケーション。社員がボタンクリックで打刻し、承認者が月次で勤怠を承認する。

## ユーザーストーリー

### 一般社員

- 社員として、出勤ボタンをクリックして出勤時刻を記録したい
- 社員として、退勤ボタンをクリックして退勤時刻を記録したい
- 社員として、自分の勤怠一覧を日別・月別で確認したい
- 社員として、自分の勤務時間と残業時間が自動計算されるのを確認したい

### 承認者

- 承認者として、部下の月次勤怠一覧を確認し、承認したい
- 承認者として、自分自身も打刻・勤怠確認ができるようにしたい

### 人事

- 人事として、全社員の勤怠を閲覧したい
- 人事として、全社員の勤怠を承認したい
- 人事として、社員マスタ（登録・編集・削除・ロール割り当て）を画面から管理したい

## 機能要件

### F-1: 認証

- ID / パスワードによるログイン
- ロールに応じた画面・機能の出し分け

### F-2: 打刻

- 出勤ボタンクリックで出勤時刻を記録
- 退勤ボタンクリックで退勤時刻を記録
- 同日に二重打刻はできない（出勤済みなら出勤ボタン無効化）

### F-3: 勤務時間の自動計算

- 勤務時間 ＝ 退勤時刻 − 出勤時刻 − 休憩1時間
- 休憩は固定1時間（12:00〜13:00）を自動控除、打刻不要
- 残業時間 ＝ 勤務時間 − 所定労働時間（7.25時間 = 7時間15分）
- 残業はマイナスにならない（所定未満は0）

### F-4: 勤怠一覧表示

- 社員: 自分の勤怠を日別・月別で表示
- 承認者: 全社員の月次勤怠一覧を表示
- 人事: 全社員の月次勤怠一覧を表示
- 表示項目: 日付、出勤時刻、退勤時刻、勤務時間、残業時間、承認ステータス

### F-5: 月次承認ワークフロー

- 承認者は月次勤怠一覧を見て、社員ごとに承認する
- 承認ステータス: 未承認 → 承認済み
- MVP では承認者は全社員の勤怠を承認可能（部署概念なし）
- 人事ロールは全社員の承認権限を持つ

### F-6: 社員マスタ管理

- 人事ロールが画面から社員の登録・編集・削除を行う
- ロール割り当て: 一般社員 / 承認者 / 人事

## ロール定義

| ロール | 打刻 | 自分の勤怠閲覧 | 他者の勤怠閲覧 | 承認 | 社員マスタ管理 |
|--------|:----:|:----:|:----:|:----:|:----:|
| 一般社員 | o | o | x | x | x |
| 承認者 | o | o | o（全員） | o（全員） | x |
| 人事 | o | o | o（全員） | o（全員） | o |

## 勤務体系

- 固定時間制: 9:15〜17:30
- 休憩: 12:00〜13:00（1時間、自動控除）
- 所定労働時間: 7時間15分（7.25時間）
- 残業: 所定超過分を残業として計算

## 技術スタック

- Backend: Java / Spring Boot
- Frontend: TypeScript / Next.js
- DB: PostgreSQL（想定）
- 認証: Spring Security（ID/パスワード）

## 利用環境

- PC ブラウザのみ（スマートフォン対応は対象外）
- 想定利用者: 約500名

## MVP スコープ外（将来対応）

- 休憩打刻
- 有給・休暇申請
- 残業事前申請フロー
- CSV / Excel エクスポート
- 外部システム連携
- レスポンス速度・可用性の SLA
- データ保持期間ポリシー
